# CLA Maintainer Notes

This file is primarily for maintainers, not general users.

This repository uses a strong inbound licensing model.

The project owner will not merge non-trivial external contributions without a signed contributor agreement on file.

Unless there is a specific reason to do otherwise, the `Project Owner` in these templates should be the company's full legal name, not the founder's personal name and not a product brand or trading name.

## Which agreement applies

Use the individual agreement if:

1. you personally own the contribution; and
2. no employer, client, university, or other organization has rights in it.

Use the corporate agreement if:

1. the contribution may be employer-owned;
2. the contribution was created in the scope of employment or contracting;
3. the contributor is submitting on behalf of a company; or
4. the maintainers request corporate confirmation.

## Project Owner Field

When you finalize these documents, replace placeholder fields with:

1. the full legal name of the company that will own the project IP;
2. the company's jurisdiction of formation when the template asks for it; and
3. the legal contact mailbox used for agreements and future diligence.

If the repository is temporarily hosted under a founder's personal GitHub or Gitea account, that does not require the project owner to be an individual. Repository hosting location and IP ownership can be different.

## Operational process

1. Contributor requests the correct CLA form.
2. Contributor or company signs and returns the agreement.
3. Maintainers record the signer, date, contact email, and covered usernames.
4. Maintainers confirm in the PR that the CLA is on file.
5. Only then can the PR proceed to normal code review.

## Recommended signing workflow

For production use, maintain signed PDF or e-signature records outside the repository and retain:

1. legal name of signer;
2. email address;
3. date of signature;
4. GitHub or Gitea username;
5. company name, if any; and
6. evidence of signer authority for corporate agreements.

Unless you later automate signature collection, provide contributors with a private legal contact mailbox such as `ivan@warded.me` and keep the final signed records outside the git repository.

## Important note

These templates are drafting material for the repository and should be reviewed by counsel before being treated as final legal instruments.
